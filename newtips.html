Code Documentation Comparison

Comparing Traditional Comments vs. the "UI Instruct" Model

1. Traditional Code Comments

Instructions are mixed directly with the code, adding clutter and mixing concerns.

/*
This class manages your local and server database.
If you want to change amount, you can change 'defaultAmount' (x4).
If you want to change type, you can change 'defaultType' (y5).
*/
class DatabaseRecord {
  constructor() {
    this.amount = 4; // This is the 'x4' from the spec
    this.type = 5;   // This is the 'y5' from the spec
    this.timestamp = new Date().getTime(); // This is '2t'
  }
}

/*
This function finds records. (find(all)0)
'status === 0' is the primary filter.
If status is 0, it processes with 'filterA' (recs2),
otherwise 'filterB' (recs).
*/
function findRecords(records, status) {
  if (status === 0) {
    // find(all)0 if 0, then recs2...
    return records.filter(rec => rec.type > 10); // "recs2"
  } else {
    // ...else recs
    return records; // "recs"
  }
}

2. "UI Instruct" Model (Hover me!)

Code is kept clean. Help is provided contextually in the UI, right where the user needs it.

? this manages your local and server database, if you want to change amount you can change x4, if you want to change type change y5 
// Pseudocode: class (recs) = x4,y5+2t
class DatabaseRecord {
  constructor() {
    this.amount = 4;
    this.type = 5;
    this.timestamp = new Date().getTime();
  }
}

// Pseudocode: find(all)0 if 0, then recs2 else recs
function findRecords(records, status) {
  if (status === 0) {
    return records.filter(rec => rec.type > 10);
  } else {
    return records;
  }
}
